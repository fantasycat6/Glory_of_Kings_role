# 数据库迁移功能使用说明

## 📖 功能概述

数据库迁移功能允许您将旧版本的数据库文件迁移到新系统中，**不会丢失任何数据**。

## 🎯 主要功能

1. **验证旧数据库** - 检查数据库文件格式和内容是否有效
2. **预览迁移数据** - 在执行迁移前查看将要迁移的数据内容
3. **执行迁移** - 将旧数据库中的数据迁移到新数据库

## 🔄 迁移模式

### 1. 合并模式（merge）
- **推荐使用**
- 保留现有数据库中的所有数据
- 只添加旧数据库中存在但新数据库中不存在的数据
- 不会删除或覆盖任何现有数据

### 2. 替换模式（replace）
- **谨慎使用**
- 清空现有用户和账号数据
- 完全使用旧数据库中的数据
- 英雄和皮肤基础数据不会被清空

## 📊 可迁移的数据

| 数据类型 | 说明 | 迁移状态 |
|---------|------|---------|
| 用户账户 | 所有用户信息 | ✅ 完全支持 |
| 游戏账号 | 用户的游戏账号（QQ/微信） | ✅ 完全支持 |
| 区服信息 | 账号下的区服 | ✅ 完全支持 |
| 英雄拥有记录 | 用户标记的已拥有英雄 | ✅ 完全支持 |
| 皮肤数据 | 皮肤基础信息 | ✅ 完全支持 |
| 皮肤拥有记录 | 用户标记的已拥有皮肤 | ✅ 完全支持 |

## 🚀 使用步骤

### 方式一：通过管理后台（推荐）

1. **登录管理后台**
   - 使用管理员账号登录
   - 进入"备份管理"页面

2. **上传旧数据库文件**
   - 点击"数据库迁移"选项卡
   - 选择您的旧版本数据库文件（.db格式）
   - 系统会自动验证数据库

3. **预览数据**
   - 查看将要迁移的数据统计
   - 确认用户名、账号、区服等信息

4. **执行迁移**
   - 选择迁移模式（推荐合并模式）
   - 点击"开始迁移"按钮
   - 等待迁移完成

5. **验证结果**
   - 检查迁移统计信息
   - 确认所有数据都已迁移

### 方式二：命令行迁移

```bash
# 进入项目目录
cd Glory_of_Kings_role

# 使用迁移脚本
python -c "
from database_migration import migrate_from_old_database
from app import app

result = migrate_from_old_database(
    app, 
    'path/to/old_database.db', 
    mode='merge'  # 或 'replace'
)
print(result)
"
```

## ⚠️ 重要提示

### 1. 备份当前数据库
在执行迁移前，建议先创建数据库备份：
- 进入管理后台 → 备份管理 → 创建数据库备份
- 或下载当前数据库文件

### 2. 合并模式 vs 替换模式
- **新用户或空数据库**：两种模式都可以
- **已有数据的数据库**：**强烈推荐使用合并模式**
- **替换模式**：会清空现有用户数据，请谨慎使用

### 3. 数据库兼容性
- 支持从 v1.0 起的所有版本
- 自动检测数据库版本和结构
- 处理字段缺失等兼容性问题

### 4. 数据完整性
- 迁移过程中自动验证数据完整性
- 自动过滤无效数据（如不存在的英雄）
- 记录迁移错误和警告信息

## 🔍 故障排除

### 问题1：验证失败
**原因**：数据库文件格式不正确或文件损坏
**解决方案**：
- 确认文件是 SQLite 数据库文件（.db扩展名）
- 尝试重新下载数据库文件
- 检查文件是否完整

### 问题2：迁移后数据不完整
**原因**：使用了替换模式，覆盖了现有数据
**解决方案**：
- 使用合并模式重新迁移
- 从备份恢复现有数据

### 问题3：部分数据迁移失败
**原因**：旧数据库中存在无效数据（如不存在的英雄）
**解决方案**：
- 这是正常现象，系统会自动跳过无效数据
- 检查迁移日志中的错误信息

## 📝 API 接口

### 1. 验证数据库
```
POST /api/admin/migration/validate
Content-Type: multipart/form-data

file: 数据库文件
```

### 2. 预览迁移数据
```
POST /api/admin/migration/preview
Content-Type: multipart/form-data

file: 数据库文件
```

### 3. 执行迁移
```
POST /api/admin/migration/execute
Content-Type: multipart/form-data

file: 数据库文件
mode: merge 或 replace
```

## 🎓 技术细节

### 数据库结构适配
迁移器会自动检测并适配以下结构差异：
- 字段名称差异
- 字段类型差异
- 缺失字段的默认值处理
- ID映射（自动重新生成新ID）

### 数据验证
- 验证英雄名称是否存在
- 验证账号和区服的关联关系
- 检查重复数据并自动跳过

### 错误处理
- 详细的错误日志记录
- 部分失败不影响其他数据迁移
- 提供详细的迁移统计信息

## 💡 最佳实践

1. **定期备份**：定期创建数据库备份
2. **测试迁移**：先在测试环境中验证迁移
3. **检查日志**：迁移后查看详细的日志信息
4. **分批次迁移**：大量数据时建议分批次迁移
5. **验证完整性**：迁移后验证数据完整性
