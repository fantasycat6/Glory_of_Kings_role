# 🚀 数据库迁移功能 - 快速使用指南

## ✅ 功能已就绪

数据库迁移功能已经成功添加到您的系统中！

## 📝 如何使用

### 1. 启动应用
```bash
cd Glory_of_Kings_role
python app.py
```

### 2. 访问管理后台
- 打开浏览器访问：http://127.0.0.1:5000
- 使用管理员账号登录
- 点击顶部导航的"管理后台"

### 3. 进入备份管理页面
- 在管理后台菜单中点击"备份管理"

### 4. 找到"数据库迁移"选项卡
- 页面会显示一个带有紫色渐变背景的"数据库迁移"卡片

### 5. 使用迁移功能
1. **上传旧数据库文件**
   - 点击"浏览"按钮
   - 选择您的旧版本 .db 数据库文件
   - 系统会自动验证数据库

2. **查看预览信息**
   - 验证通过后，会显示数据统计
   - 包括用户、账号、区服、英雄记录等数量

3. **选择迁移模式**
   - **合并模式（推荐）**：保留现有数据，只添加新的
   - **替换模式**：清空现有数据，完全使用旧数据

4. **执行迁移**
   - 点击"开始迁移"按钮
   - 等待迁移完成
   - 查看迁移结果统计

## ⚠️ 重要提示

### 1. 备份当前数据库
在执行迁移前，建议先创建数据库备份：
- 在"数据库备份"卡片中点击"创建数据库备份"

### 2. 合并模式 vs 替换模式
- **新用户或空数据库**：两种模式都可以
- **已有数据的数据库**：**强烈推荐使用合并模式**
- **替换模式**：会清空现有用户数据，请谨慎使用

### 3. 迁移后验证
- 检查迁移统计信息
- 确认所有数据都已迁移
- 测试登录功能是否正常

## 🔍 故障排除

### 问题1：上传文件后没有反应
**解决方案**：
- 检查文件扩展名是否为 .db
- 检查文件是否损坏
- 查看浏览器控制台错误信息

### 问题2：验证失败
**解决方案**：
- 确认是 SQLite 数据库文件
- 尝试重新下载数据库文件
- 检查文件是否完整

### 问题3：迁移后数据不完整
**解决方案**：
- 使用合并模式重新迁移
- 从备份恢复现有数据
- 查看迁移日志中的错误信息
